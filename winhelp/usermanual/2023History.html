Version 2.3.7.6          September 24, 2023               Build 1619

 

     


The option to install all Grub2Win modules to the EFI partition has been removed.


This option was available as a workaround in cases where the install partition was encrypted.

It was rarely used.


Over time, this method has proved unreliable and caused many boot problems.


From now on, Grub2Win must be installed to an unencrypted partition.



Note: Your Grub2Win EFI partition modules will be refreshed to level 154.




Added the Haiku BFS filesystem to the partition scan code.

Version 2.3.7.2          July 1, 2023                    Build 1610

 

     


Experimental support for ChromeOS.

This requires the installation of Brunch available from Github.


After running the Brunch chromos-install.sh command

you must use Grub2Win to import the Brunch generated configuration file.

This file is usually named chromeos.img.grub.txt


Here is a helpful YouTube Chrome tutorial by Tricknology.

Use the new Grub2Win Chrome import process.


https://www.youtube.com/watch?v=OicSb2QpAVk



Note: Any support requests for ChromeOS boot and configuration

problems must be made to the Chrome or Brunch support forums.




Fixed an issue caused by uppercase UUID in the generated Linux menu code.

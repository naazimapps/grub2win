Version 2.3.4.1        July 25, 2022




Added a dialog to verify that the Grub2Win boot menu appears.


Some EFI firmware does not properly follow standards at boot time.

It skips over the Grub2Win kernel and loads Windows instead.

When this happens, the Grub2Win boot menu never appears.


If requested, the new dialog will display a help file.

The help file contains detailed instructions to remedy the problem.

Version 2.3.3.6        May 28, 2022




Removed the “Additional Drivers” option for BIOS systems.


These old GNU Grub drivers were very rarely used.


They did not work at all on EFI firmware systems.

They sometimes caused boot time lockups on BIOS systems.


Any users who actually need the old drivers, should contact me at SourceForge.

I will set up a workaround for you.
